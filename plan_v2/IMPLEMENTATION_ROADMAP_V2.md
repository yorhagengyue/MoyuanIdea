# Moyuan AI-Native Cultural Education System
## Implementation Roadmap V2
更新时间：2026-03-30

---

## 1. 路线图原则

这次路线图不再按“想到什么做什么”推进。
统一改成三层：

1. 先立住基础骨架
2. 再拉出孩子端可见爆点
3. 最后扩张文化内容和协作层

也就是说：
第一阶段解决“系统能不能站住”
第二阶段解决“孩子和家长会不会被打动”
第三阶段解决“品牌辨识度和长期壁垒”

---

## 2. Phase 1：基础骨架落地（0-8 周）

### 2.1 目标
让系统第一次形成完整闭环：
课堂 -> 作品 -> 证据 -> 家长可见 -> 课后延续

### 2.2 必须交付
#### A. 作品中台最小版
- ArtworkSnapshot
- ProjectCycle 基础版
- 作品阶段状态
- 作品与 session 绑定

#### B. 课堂执行系统最小版
- LessonPack
- LessonSession
- teacher control
- 点名 / 开课 / 结束
- 基础 fallback

#### C. 成长卷轴最小版
- 出勤状态
- 作品快照
- 老师一句点评
- proud moment
- next preview

#### D. 老师副驾 AI
- 家长沟通草稿
- 课后反馈草稿
- 基础课后任务草稿

#### E. 家长端最小版
- 日历式课程视图
- 今日作品
- 今日变化
- 课后简反馈

#### F. 课堂 AI 演示点
从已有 phase2_interactive_lesson 与 phase3_hanzi_action_lesson 中，至少跑通一个完整闭环 demo

### 2.3 成功标准
- 老师能在低操作负担下完成一节课的闭环记录
- 家长端不再只是通知，而是能看到“作品 + 变化 + 下次”
- 课堂 AI 能形成一个稳定可演示样板
- 作品能开始以 project cycle 方式沉淀

---

## 3. Phase 2：孩子端核心爆点（8-16 周）

### 3.1 目标
让课后真的持续发生，让孩子端不再只是附属物

### 3.2 必须交付
#### A. AI 学伴成长系统基础完整版
- CompanionProfile
- CompanionSession
- DiaryEntry
- 半开放陪伴式对话
- 对话收束机制

#### B. SkillMastery 与统一成长地图
- 开口表达线
- 作品讲述线
- 文化理解线
- 复习延续线

#### C. SkillCard / Achievement
- 稀缺发放
- 以连续达标为准
- 系统自动审核为主，老师只抽检异常

#### D. 声音成长系统优先子线
优先顺序：
1. 诗词旋律化
2. 单人故事角色配音

#### E. 成语谚语小剧场基础版
- curated knowledge library
- 模板动画骨架
- 角色参与但不失控
- 动画后复述与应用

### 3.3 成功标准
- 孩子愿意在课后主动回来至少完成轻互动
- 家长开始看到“孩子今天说了什么、会了什么”
- 技能卡不再像玩具，而开始具有成长感
- 角色系统真正承载学习，不是闲聊玩偶

---

## 4. Phase 3：高辨识度文化层（16-28 周）

### 4.1 目标
让系统从“做得通”升级到“这就是 Moyuan”

### 4.2 必须交付
#### A. 色彩源流系统
- 6 个代表性色彩
- color story card
- color reflection
- 与作品和家长端绑定

#### B. 作品声景成长线
- 一件作品对应声音身份
- project cycle 不同阶段声音变化
- portfolio 中保留声音轨迹

#### C. 双人小剧场配音系统
- child + AI
- child + child（异步）
- narrator mode
- duet scene evidence

#### D. 角色沉浸层升级
- 半 3D / 3D 增强
- 更稳定动画动作
- 不把 3D 当主引擎

### 4.3 成功标准
- 内容模块形成 Moyuan 独有气质
- 家长能明显感知“这不是普通培训机构软件”
- Portfolio 开始具有图、文、声、角色演绎等多模态价值
- 系统开始具备更强展示性和长期壁垒

## 6. 当前最不该做的事情

### 6.1 不要同时铺太多孩子端模块
角色、诗词、颜色、双人剧场、成语动画，如果同时重做，很容易一起拖死。
必须按主线优先级分批。

### 6.2 不要让老师承担审核中心
老师只能做抽检和点火，不应该成为大量 AI 结果的逐条审核者。

### 6.3 不要急着把 3D 做成卖点中心
3D 有吸引力，但它不是第一推动力。
真正的第一推动力仍然是：
作品 + 表达 + 成长证据

### 6.4 不要把家长端继续做成信息分发器
家长端如果继续是：
- 通知
- invoice
- 打卡
这套系统会退回旧思路。

## 7. 每一阶段的对外讲法

### Phase 1 对外
“我们先把机构从人工和碎片化流程里拉出来，并让家长第一次持续看到作品与变化。”

### Phase 2 对外
“我们让孩子课后继续和作品、角色、语言发生关系，而不是只在课堂里短暂发生学习。”

### Phase 3 对外
“我们开始形成 Moyuan 自己独特的文化学习语言：声音、颜色、小剧场、角色演绎全部被结构化。”

## 8. 内部推进顺序建议

### 优先级 A：立即推进
- 作品中台
- 课堂执行系统
- 成长卷轴
- 家长端最小可感知版本
- 老师副驾 AI

### 优先级 B：尽快推进
- AI 学伴成长系统
- 技能掌握度与技能卡
- 诗词旋律化
- 成语谚语小剧场

### 优先级 C：品牌增强
- 色彩源流系统
- 双人小剧场配音
- 作品声景成长线
- 高阶 3D 角色

---

## 9. 最终要求

后续任何新文档、新页面、新接口、新 pitch，都必须先回答这四个问题：
1. 它回到哪一个中心对象（作品 / evidence / scroll / portfolio）？
2. 它属于哪一层 AI（老师 / 课堂 / 家长 / 未来独立学习）？
3. 它发生在课前 / 课中 / 课后 / 长期的哪一个位置？
4. 它有没有让系统更统一，而不是更分散？

答不上来，就不该优先做。
