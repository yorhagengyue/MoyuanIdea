# Moyuan AI-Native Cultural Education System
## MODULE_ACCEPTANCE_CRITERIA_V2
更新时间：2026-03-30

---

## 1. 这份文档的作用

V2 已经有：

- 总母版
- 路线图
- 模块目录

但如果没有统一验收标准，团队仍然会在这些地方走偏：

- 觉得“做出来了一个页面”就算模块落地
- 觉得“AI 能回答”就算模块成立
- 觉得“孩子说了几句”就算课后系统成立
- 觉得“发了很多卡片”就算成长地图成立

这份文档的作用是把每个关键模块的 **准入标准 / 完成标准 / 禁止事项 / 成功指标** 一次写清。

---

## 2. 总体准入标准

任何模块要进入 V2 主计划，必须先回答这 6 个问题：

| 问题 | 要求 |
|---|---|
| 服务谁 | 必须明确是家长 / 老师 / 管理端 / 孩子中的谁 |
| 发生在哪 | 必须明确课前 / 课中 / 课后 / 长期中的哪一段 |
| 回到哪里 | 必须明确最终回到作品 / evidence / scroll / portfolio 中哪一个 |
| 属于哪层 AI | 必须属于老师副驾 / 课堂共创 / 家长沟通 / 未来独立学习之一 |
| 怎么不打散系统 | 必须说明如何和主链对齐，而不是另起世界 |
| 怎么验收 | 必须定义“做成”与“没做成”的边界 |

如果答不上来，就不进入主计划。

---

## 3. V2 总体成功标准

系统整体并不是“功能越多越好”，而是要证明 4 件事：

1. 课堂可以稳稳跑完  
2. 家长能持续看到作品与变化  
3. 孩子课后愿意继续发生表达  
4. 长期开始形成 portfolio 与 project cycle 资产  

这 4 条如果站不住，再多模块都只是壳。

---

## 4. 基础骨架模块验收

---

## 4.1 作品中台

### 模块目标
统一承载作品、阶段、颜色理解、声音轨迹、老师点评等对象，成为所有孩子端模块的中心落点。

### 必须具备
- `ArtworkSnapshot`
- `ProjectCycle`
- 作品阶段状态
- 作品与 session 绑定
- 作品与 portfolio 的可沉淀关系

### 完成标准
- 老师每节课至少能留下作品快照
- 多节课的作品能被串回同一个 `ProjectCycle`
- 家长端能看到阶段推进，而不是一张孤立图片
- 孩子端模块产出的内容能回绑到作品

### 失败信号
- 作品只是被当成一张上传图片
- 不同课的作品没有长期关系
- 家长端看不到“做到哪了”
- 声音、颜色、对话结果无法回绑作品

### 关键指标
- 作品快照产出率
- 作品快照与 session 绑定率
- project cycle 建立率
- portfolio entry 生成率

---

## 4.2 课堂执行系统

### 模块目标
让老师能低负担地完成真实课堂控制，课堂 AI 必须始终被老师掌控。

### 必须具备
- LessonPack
- LessonSession
- teacher control
- turn / attempt
- fallback
- report

### 完成标准
- 老师可以一键开课
- 当前节点始终由老师端控制
- 噪音或弱网场景下可以 fallback
- 一节课结束能形成 report
- 学生端只承担简单互动，不承担课堂推进

### 失败信号
- 学生端状态漂移，老师无法稳定控制
- AI 发散导致课堂节奏被拖慢
- 弱网一来就中断课堂
- 老师为了系统不停点按钮

### 关键指标
- 课堂完成率
- degraded/manual 触发率
- 老师单节点额外操作次数
- AI 响应 P95
- fallback 成功率

---

## 4.3 成长卷轴系统

### 模块目标
把 raw evidence 编排成家长能理解的一节课结果。

### 必须具备
- 出勤
- 作品
- proud moment
- 老师一句点评
- 今日变化
- 下次预告

### 完成标准
- 家长打开后能 5 秒内理解今天发生了什么
- 一节课至少能形成 1 个 scroll
- scroll 内容不是 raw log，而是编排视图
- 家长能看到 next preview

### 失败信号
- scroll 只是“老师发了一段话”
- 只有通知，没有作品
- 只有图片，没有变化说明
- 家长无法理解今天到底哪里变了

### 关键指标
- scroll 发布率
- scroll 完整率
- 家长点击率 / 打开率
- 有 proud moment 的 scroll 比例

---

## 4.4 家长端成长感知层

### 模块目标
让家长端从公告栏变成成长感知入口。

### 完成标准
- 首页不以菜单为中心，而以“今天 / 本周孩子发生了什么”为中心
- 家长能直接看到作品、变化、下次
- 课后任务入口自然，不像推广位

### 失败信号
- 首页以 invoice / 通知 / 菜单为主
- 需要点很多层才能找到作品
- 家长端只在有行政消息时才有意义

### 关键指标
- 首页打开率
- 作品卡点击率
- scroll 查看率
- 课后轻任务启动率

---

## 4.5 老师副驾 AI

### 模块目标
减少老师重复写反馈、任务和沟通文案的时间。

### 必须具备
- 课后反馈草稿
- 家长沟通草稿
- 基础课后任务草稿
- draft / review / publish 机制

### 完成标准
- 草稿质量达到“老师轻改即可发”
- 不是 AI 自动乱发
- 老师只做抽检/轻审核
- 草稿能调用今天 evidence，而不是空模板

### 失败信号
- 草稿泛泛而谈
- 老师要大改
- 敏感内容直接自动发
- 和今天课堂无关

### 关键指标
- 草稿通过率
- 老师编辑平均字数
- 审核耗时
- 误发率 / 退回率

---

## 5. 孩子端核心模块验收

---

## 5.1 AI 学伴成长系统

### 模块目标
让孩子课后继续围绕今天课堂和作品发生表达，而不是自由陪聊。

### 必须具备
- CompanionProfile
- CompanionSession
- warm_share -> guided_reflection -> celebrate -> close
- DiaryEntry
- 与今日 evidence 绑定的对话任务

### 完成标准
- 孩子愿意回来进行轻互动
- 对话允许轻自由分享，但能温柔收束回学习主线
- 系统能围绕今天 evidence 提问，而不是泛聊天
- 最终形成 diary / scroll / skill 更新

### 失败信号
- 角色变成普通聊天玩偶
- 孩子越聊越偏
- 对话与今天课堂无关
- 日记只是聊天记录转储

### 关键指标
- 课后返回率
- guided_reflection 完成率
- diary 生成率
- 与 source_session 绑定率

---

## 5.2 SkillMastery / 技能地图

### 模块目标
让孩子端成长有稳定结构，而不是轻浮激励。

### 必须具备
- 统一成长地图四条主干
- SkillMastery
- SkillCard
- Achievement
- 连续达标逻辑

### 完成标准
- 技能节点不是一次触发即发卡
- 普通节点以连续达标为通过线
- milestone / achievement 稀缺
- 技能卡都能追溯到 evidence

### 失败信号
- 聊一会就发卡
- 用时长、签到替代真实学习表现
- 卡片太多太轻
- 老师需要人工逐张审核

### 关键指标
- card 发放稀缺度
- card 的 evidence 可追溯率
- 连续达标通过率
- 家长对 skill card 的理解度

---

## 5.3 角色沉浸层

### 模块目标
增强归属感与持续使用意愿，让 AI 被直接看见。

### 必须具备
- 2D / 半 3D 外显
- companion avatar 出现在对话 / 日记 / skill / 剧场
- 角色风格受控
- 可选 3D，但不是强依赖

### 完成标准
- 角色增强沉浸，不抢主线
- 角色存在感清晰，但不会压过学习内容
- 3D 只是增强，不影响基础体验

### 失败信号
- 全部注意力被 3D 外观夺走
- 角色视觉很强，但对学习没有帮助
- 生成门槛太高，孩子或家长难以完成

### 关键指标
- 角色创建完成率
- 角色复用率
- 对话启动率提升
- 3D 相关失败率

---

## 5.4 声音成长系统

### 模块目标
通过诗词旋律化、故事配音、作品声景，让文化内容更易进入、更易记住、更愿意重复。

### 必须具备
- 诗词旋律化
- 故事角色配音
- 作品声景成长线（后续可增强）
- 声音 evidence 进入 scroll / portfolio

### 完成标准
- 音乐服务于记忆与表达，不替代表达
- 孩子会跟、会说、会录，而不只是听
- 声音任务能回到今天课题
- 声音结果能形成家长可见 evidence

### 失败信号
- 变成自由音乐生成玩具
- 孩子只听不说
- 声音和今天课堂无关
- 家长端看不懂声音结果的学习价值

### 关键指标
- 跟读/配音完成率
- 音频 evidence 发布率
- 复听率 / 重复率
- 与 source_session 对齐率

---

## 5.5 成语谚语小剧场

### 模块目标
让抽象词汇和文化点变得有画面、能复述、可代入。

### 必须具备
- curated knowledge library
- 模板动画骨架
- 角色受控参与
- 看完后复述 / 应用

### 完成标准
- 小剧场短、小、低龄可懂
- 成语不被 personalization 弄坏原典
- 动画后一定回到孩子开口
- 结果进入 retell / diary / skill / scroll

### 失败信号
- 输入任意知识点自由生成，质量不稳
- 孩子只看动画，不说话
- 角色粗暴替换原典主角
- 内容与今天课题长期无关

### 关键指标
- 小剧场播放完成率
- 复述完成率
- 词义理解水平
- 与课题联动层匹配率

---

## 5.6 双人小剧场配音系统

### 模块目标
让两个孩子或孩子 + AI 在受控小剧场里协作完成一个知识点任务。

### 必须具备
- child + AI
- child + child（异步）
- narrator mode
- duet_scene evidence

### 完成标准
- 不做开放社交
- 不做自由实时聊天
- 使用模板场景和角色分配
- 最终生成一个可回看的小剧场结果

### 失败信号
- 走向实时语音房
- 孩子需要长时间等待对方
- 知识点被协作形式压掉
- 家长看不懂完成了什么

### 关键指标
- 场景完成率
- 录音合成成功率
- 异步协作完成率
- duet_scene evidence 产出率

---

## 5.7 色彩源流系统

### 模块目标
让孩子通过颜色进入中国文化，并把颜色说回自己的作品。

### 必须具备
- ColorStoryCard
- ColorReflection
- 6 个代表性色彩
- 颜色来源、感觉、用途、回到作品的追问

### 完成标准
- 不走复杂工艺史
- 不做危险材料体验主线
- 每次颜色内容都回到作品表达
- 可以进入 Growth Scroll 和 Portfolio

### 失败信号
- 讲很多知识，却没有作品表达
- 孩子记住“颜色来自矿物”，但不会说自己的画
- 模块独立存在，和课堂无关

### 关键指标
- color_reflection 生成率
- 孩子颜色句输出率
- 家长可见色彩理解条目率
- 与 artwork 绑定率

---

## 6. 课堂内容样板模块验收

---

## 6.1 图片互动课 AI 引擎

### 模块目标
形成课堂可演示、老师可控、孩子愿意参与的受控即兴 AI 样板。

### 当前已定义的硬指标
- 20 台设备可同时跑完 10 分钟流程
- 孩子平均互动次数 >= 2
- 老师单节点额外操作 <= 3
- AI 延迟 P95 <= 2.5 秒
- 8 分钟内能演示闭环

### 额外准入要求
- 严格受 `allowed_facts` 约束
- fallback 稳定
- teacher control 始终可用

---

## 6.2 汉字声像行动课

### 模块目标
证明知识点可以被 AI 活化成低龄可理解、可复述的内容模板。

### 完成标准
- 一个汉字只讲一个清楚点
- 动画时长可控
- 语言解释避免争议考据
- 看完后能引导孩子说出来

### 失败信号
- 动画很好看，但孩子不知道重点
- 内容不稳定、不同字风格断裂
- 讲成考据课

---

## 7. 支线模块验收

---

## 7.1 PSLE / DSA 线

### 模块定位
支线，不抢主叙事。

### 准入要求
- 必须明确高年龄段适用边界
- 不稀释作品、文化、表达主线
- 不作为品牌中心对外表达

### 失败信号
- 外部叙事又变回应试系统
- 低龄主产品被高年龄支线夺走资源

---

## 8. 课题联动层的额外验收

虽然它不是直接给用户看的模块，但必须单独验收。

### 完成标准
- 课后内容优先围绕今天课题展开
- 不能直连时，桥接路径可解释
- 选出的模块不会让内容突然断裂
- 家长能感受到课堂与课后的一致性

### 失败信号
- 课后内容随机
- 明显出现硬贴文化点
- 不同模块各讲各的
- 孩子感觉晚上换了另一门课

### 关键指标
- DailyContentPlan 生成率
- 课题-模块相关度评分
- 家长端主题一致性感知
- 被老师手动替换桥接方案的比例

---

## 9. 感官表达层的验收边界

这层不应作为独立大模块验收，而应作为“作品反思增强层”。

### 成立条件
- AI 能引导孩子用“像什么味道/声音/感觉”来描述作品
- 最终回到语言表达
- 不依赖真实嗅觉硬件

### 不成立的版本
- 老师每节课都准备实体气味瓶
- 变成卫生、安全、过敏负担很高的流程
- 与作品表达脱钩

---

## 10. 统一发布规则

### 10.1 家长可见对象的统一标准
任何要进入家长端的东西，至少满足：

- 有 `source_session_id`
- 有 `student_id`
- 有 `visibility_scope`
- 有 `review_status`
- 能被解释成今天发生的变化

### 10.2 不允许直接发 raw logs
- TurnAttempt 不直接给家长看
- 原始 ASR 不直接给家长看
- 原始聊天记录不直接给家长看

### 10.3 家长端看到的必须是编排结果
- GrowthScroll
- 已发布 Diary
- 已发布 SkillCard
- 已选中的作品与高光

---

## 11. 统一通过门槛表

| 模块 | 可以进入主计划的最低门槛 |
|---|---|
| 作品中台 | 有 ArtworkSnapshot + ProjectCycle 基础关系 |
| 课堂执行系统 | LessonSession 稳定跑通 + fallback 可用 |
| 成长卷轴 | 能稳定生成作品 + proud moment + next preview |
| AI 学伴成长系统 | 对话能收束回学习主线并产生日记 |
| SkillMastery | 连续达标逻辑成立，不轻易发卡 |
| 声音成长系统 | 至少一条子线能稳定形成声音 evidence |
| 成语谚语小剧场 | 看完后必须能复述，不只播放动画 |
| 双人小剧场 | 先做到异步 child+AI / child+child |
| 色彩源流系统 | 每次颜色内容都能回到作品表达 |

---

## 12. 一句话总结

> **V2 不是“功能做出来就算完成”，而是“每个模块都必须证明自己让作品、证据、成长感知这条主链更强”。**
