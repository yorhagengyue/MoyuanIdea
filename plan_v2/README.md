# plan_v2

这组文档用于替代仓库中旧的“管理系统 + 分散 phase 模块”叙事，作为当前版本的总母版。

建议阅读顺序：

1. `MASTER_SYSTEM_PLAN_V2.md`
   - 看系统世界观、主链、三端分离、AI 分层、中心对象与模块总定位

2. `IMPLEMENTATION_ROADMAP_V2.md`
   - 看阶段推进顺序、优先级、旧文档如何重新映射到新结构

3. `MODULE_CATALOG_V2.md`
   - 看每个模块的用途、边界、优先级与准入标准

这套文档的目的不是替代已有 phase 实施文档，而是成为它们的上位定义。
后续任何新 idea、新页面设计、新接口或新的 pitch 叙事，都应先对齐 V2 主计划，再决定是否进入仓库主线。
