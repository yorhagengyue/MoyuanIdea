## 当前系统主线

V2 之后，这个项目的统一主链是：

**课前预期建立 -> 课中作品创造与 AI 活化 -> 课后成长证据生成 -> 长期作品集与成长档案沉淀**

系统中心不再是课程表，而是：
- 作品（Artwork / Project Cycle）
- 证据（Evidence）
- 成长卷轴（Growth Scroll）
- 长期作品集（Portfolio）

系统按三端分离：
- 家长端：成长感知入口
- 老师端：执行副驾
- 管理端：机构大脑

## 这些旧文档现在怎么理解

旧文档不会删除，因为它们记录了：
- 早期方向
- 课堂样板
- 已成型内容模块
- 仍可复用的接口与课包资产

但从现在开始，**它们都应该被理解为 V2 主计划下的子模块或历史阶段产物**，而不是仓库的总叙事本身。

也就是：
- `plan_v2/` 是新的总入口
- 原有 phase 文档是具体模块或实施样板
