# 课堂受控 AI Demo 实施规格

## 1. 想让老板看到什么
我们把“图片互动课”从概念变成可上课的流程。
老师能控节奏，孩子愿意参与，系统在弱网和噪音下也不崩。

## 2. 最小流程（10分钟）
导入 1 分钟 -> 观察 3 分钟 -> 表达 3 分钟 -> 巩固 2 分钟 -> 总结 1 分钟。
主题先固定“山的不同叫法”。

## 3. 页面需求
### 老师主控页
要有：开课、下一步、重问、fallback、暂停。
要看得到：当前节点、加入人数、网络状态、本节点 fallback 次数。

### 学生互动页
一屏只做一件事：看图+回答。
输入仅两种：按住说话 / 点选按钮。

## 4. 受控规则（必须硬编码）
- AI 只能使用当前节点的 allowed_facts。
- 回复最多两句短句。
- 追问最多两轮。
- 识别置信度低于 0.65 或响应超 2.5 秒，自动 fallback。

## 5. 演示时最好用的真实例子
孩子说“这是峰，因为它尖尖的”。
系统回应“对，尖高像峰。那你再找一个更像岭的吗？”
如果识别不稳，直接出现“峰/岭/崖”按钮，孩子点一个继续。

## 6. 你们要完成的具体产物
- 一条可跑通 LessonSession 流程
- 一套固定课包（已存在，可直接复用）
- 一份课堂结果面板（参与率+典型表达+下节建议）

## 7. 验收
- 20 台并发能完成完整流程
- 老师单节点额外操作 <= 3
- P95 响应 <= 2.5 秒
